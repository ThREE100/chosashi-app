# 公開手順（外出先のiPhoneから使う）

このアプリは **PWA対応済み** です。一度ネットに公開すれば、iPhoneのSafariでURLを開き
「ホーム画面に追加」すると、**アプリのようにアイコンから起動**でき、**電波が悪くても動きます**。

公開する中身は `dist` フォルダ（`npm run build` で生成）です。

---

## ステップ1：ビルドする（最新の状態を書き出す）
PowerShell でこのフォルダに入って：
```
npm run build
```
→ `dist` フォルダが最新になります。アプリを更新するたびに、この後の公開をやり直せばOK。

---

## ステップ2：ネットに公開する（どちらか一方でOK）

### 方法A：Netlify Drop（いちばん簡単・ターミナル不要）
1. ブラウザで **https://app.netlify.com/drop** を開く
2. このフォルダ内の **`dist` フォルダ** を、ページにドラッグ＆ドロップ
3. すぐに `https://○○○.netlify.app` というURLが発行される
4. （URLを残すには無料アカウント登録。未登録だと数時間で消えます）

### 方法B：Cloudflare Pages（前回の続き・URLが固定でおすすめ）
PowerShell で：
```
npm run deploy
```
- 初回はブラウザが開いてCloudflareログインを求められます（1回だけ）
- 完了すると `https://chosashi-app.pages.dev` のような固定URLが出ます
- 2回目以降も `npm run deploy` だけで更新を反映できます

> どちらも無料。`dist` を公開するという点は同じです。

---

## ステップ3：iPhoneでアプリとして使う
1. iPhoneの **Safari** で、発行されたURLを開く
2. 下の **共有ボタン（□に↑）** をタップ
3. **「ホーム画面に追加」** を選ぶ
4. ホーム画面にアイコンができる → タップで全画面起動・オフラインでも動作

---

## よくある質問
- **PCを消しても見られる？** → はい。公開後はあなたのPCと無関係にネット上で動きます。
- **他人に見られたくない** → URLを知っている人だけがアクセスできます（検索には出にくい）。
  本格的に隠すならCloudflare Pagesの «アクセス制限» 機能などで保護可能。
- **問題を追加したら？** → ステップ1（ビルド）→ ステップ2（公開）をやり直すだけ。
  iPhone側はアプリを開けば自動で最新に更新されます（PWAの自動更新設定済み）。
